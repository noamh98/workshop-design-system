מדריך פורטבילי, לא תלוי-כלי: עובד מול Claude Code, Claude.ai, Copilot 365,
ChatGPT, או כל סוכן אחר. אין build step ואין ספריות — זה HTML/CSS/JS רגיל,

## שלב 0 — הבנת המהות (חשוב לפני שממשיכים)

המערכת הזו מייצרת **קובצי HTML עצמאיים** שנפתחים בדפדפן (`file://`, בלי
שרת) ומודפסים ל-PDF במידת הצורך — כל שקף הוא `1920x1080`. **זו לא מצגת
PowerPoint/.pptx.** אם אתה עובד מול Copilot 365 ומצפה לקובץ `.pptx` שנפתח
ב-PowerPoint — זה צינור עבודה אחר לגמרי ולא תואם למערכת הזו. Copilot יכול
בהחלט לעזור לך לכתוב/לערוך את קובצי ה-HTML עצמם (יש לו גישה לקבצים בריפו
אם תיתן לו), אבל התוצר הסופי נשאר HTML שנצפה בדפדפן, לא שקופית PowerPoint
native.

## שלב 1 — מה צריך להיות פיזית זמין לסוכן

כל סוכן — Claude, Copilot, GPT, לא משנה — צריך גישה לארבע התיקיות האלה
מהריפו `workshop-design-system`:

```
css/      — טוקנים, מערכת דיו, style-sketchbook.css, fonts-sketch.css
js/       — sketch.js (קווי-יד), charts.js, theme.js
fonts/    — Excalifont + Playpen Sans Hebrew, self-hosted
icons/    — sprite.svg (מקור קנוני לכל האייקונים, כולל הדואטון של KPI)
```

שתי דרכים לספק את זה:

1. **גישה לריפו** — תן לסוכן קישור/גישה ל-`noamh98/workshop-design-system`
   ותגיד לו לעבוד מולו ישירות (זה מה ש-Claude Code עושה עם `add_repo`).
2. **העתקה מקומית** — אם הסוכן שלך לא יכול לגשת ל-git (למשל Copilot Chat
   בהקשר שאין לו גישת ריפו), **תוריד/תעתיק** את ארבע התיקיות למעלה +
   קובץ template אחד לדוגמה (`templates/sketch-dashboard.he.html` או
   `templates/finetracker-briefing.he.html`) לפרויקט המקומי שלך, ותכוון
   את הסוכן לעבוד שם.

בלי הקבצים האלה בפועל (לא רק "לדעת שהם קיימים") — שום סוכן לא יכול לייצר
פונטים/צבעים/קווי-יד תואמים. זו לא "ידע כללי", זה asset ממשי.

## שלב 2 — הפרומפט שנותנים לסוכן (מוכן להעתקה)

תעתיק את הבלוק הזה לתוך השיחה עם כל סוכן (Claude, Copilot, GPT...), אחרי
שנתת לו גישה לקבצים משלב 1:

```
אני רוצה קובץ HTML חדש, עצמאי (1920x1080), באותו סגנון "sketchbook" של
workshop-design-system — נייר בז' עם גרעיניות, מסגרת פחם כהה, פונט כתב-יד
(Excalifont ללועזית / Playpen Sans Hebrew לעברית דרך css/fonts-sketch.css),
כרטיסים עם קו-יד רועד אמיתי (לא CSS border), אייקונים בצבע-יד.

קח כדוגמת-בסיס את templates/sketch-dashboard.he.html (או
finetracker-briefing.he.html למצגת רב-שקפית) — תעתיק את מבנה ה-<head>
(את כל ה-<link> לקבצי css/ וה-<script> לקבצי js/), את מוסכמת שש-הדיו
(--ink-black/blue/red/green/orange/purple דרך data-ink, בלי לצבוע בעצמך
בערכים קשיחים), ואת קלאסי הרכיבים הקיימים: .sketch-card, .sketch-list
(--bullets / --checks), .sketch-flow + data-sketch="circle"/"arrow-h",
.sketch-kpi. אל תמציא מערכת עיצוב חדשה — תשתמש במה שכבר קיים.

התוכן שאני רוצה במצגת: [התוכן שלך כאן]

בעברית: כל טקסט אנגלי בתוך משפט עברי עטוף ב-<bdi class="ltr-embed">,
ומספרים/אחוזים ב-<bdi class="tabular">, כדי שה-bidi לא יתהפך.

חובה: תרנדר בדפדפן ותצלם מסך לפני שאתה אומר שזה מוכן — אל תניח שזה נראה
טוב בלי לראות פיקסלים.
```

## שלב 3 — כללי ברזל (שלא משתנים בין סוכנים)

- **שש דיו קבועות, לא יותר**: שחור/כחול/אדום/ירוק/כתום/סגול. אף פעם לא
  ממציאים צבע שביעי, אף פעם לא ממפים מחדש את המשמעות (אדום=בעיה, כחול=עתיד
  וכו') — ראו `css/ink-system.css`.
- **אין CDN לפונטים של ה-sketchbook**: הכול דרך `css/fonts-sketch.css`
  שמצביע ל-`fonts/*.woff2` יחסית. אם רואים `fonts.googleapis.com` בקוד של
  שקף sketchbook חדש — זו טעות.
- **קווי-יד אמיתיים, לא CSS border מזויף**: `data-sketch="box"` /
  `"circle"` / `"arrow-h"` על אלמנט + `js/sketch.js` בעמוד. ה-CSS border
  הוא רק גיבוי לרגע לפני שה-JS רץ.
- **bidi בעברית**: `<bdi class="ltr-embed">` סביב כל ריצת טקסט לועזי בתוך
  משפט עברי, `<bdi class="tabular">` סביב מספרים/אחוזים.
- **render-verify**: כל שינוי צריך צילום מסך אמיתי (Chromium headless אם
  יש, או פשוט פתיחה בדפדפן) לפני שמכריזים "מוכן". זו בדיוק הסיבה שהגרסה
  הקודמת של השקף הזה נכשלה בפעם הראשונה.

## שלב 5 — מסירה כקובץ עצמאי (ברירת מחדל כשמבקשים "קובץ HTML")

הקבצים ב-`templates/` טוענים `css/js/fonts/` דרך נתיבים יחסיים (`../css/...`)
— זה מעולה לפיתוח בתוך הריפו, אבל לא נוח אם רוצים **לשלוח קובץ אחד** למישהו
או לפתוח אותו מחוץ למבנה התיקיות. **כשמשתמש מבקש "שלח לי את המצגת ב-HTML"
(ולא ציין אחרת) — התוצר הנכון הוא קובץ עצמאי אחד**, לא הקובץ המקושר.

יש לזה כלי מוכן, `scripts/bundle-standalone.py` — סקריפט Python טהור
(ללא pip install, תואם לפילוסופיית "בלי תלויות" של הריפו) שאורז קובץ
template לקובץ HTML יחיד: כל ה-CSS וה-JS מוטמעים inline, וכל הפונטים
מוטמעים כ-base64 data URI. אפס בקשות רשת, אפס תלות בנתיבים יחסיים:

```bash
python3 scripts/bundle-standalone.py templates/sketch-dashboard.he.html
# כותב templates/sketch-dashboard.he.standalone.html
```

**חובה לאמת render אחרי האריזה** — פתח את התוצר בדפדפן (או Chromium
headless עם `--host-resolver-rules="MAP * ~NOTFOUND"` כדי לוודא שהוא
באמת אופליין) ותוודא שהניווט בין שקפים (אם יש) עדיין עובד ושלא נוצר
`<script>` מקונן בטעות — זו בדיוק התקלה שנתפסה בפעם הראשונה שהסקריפט
הזה נכתב, ולכן הוא בנוי על regex לפי תוכן ולא לפי מספרי שורות.
