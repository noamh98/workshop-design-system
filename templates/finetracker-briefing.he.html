1 · מצב קיים — לפני המערכת

ניהול ידני, ללא מעקב ובקרה

לפני FineTracker דוחות התנועה וההסבות שלהם נוהלו ידנית — ללא מקור אמת יחיד, ללא תיעוד וללא בקרה על מצב הטיפול.

אין מעקב דוחות נרשמו ידנית ב-Excel — לא ידוע איזה דוח בטיפול, נדחה או ממתין להסבה.
אין תיעוד מסמכי הדוח וההסבות מפוזרים במיילים ובתיקיות רשת — ללא קישור לדוח.
אין בקרה אין נראות לסטטוס ההסבה לנהג, ואין דרך לוודא שכל דוח טופל עד הסוף.
דוחות "נופלים" קנסות באיחור, קנסות כפולים ותשלומים מיותרים — בגלל היעדר מעקב.
אין Audit אי אפשר לדעת מי שינה מה ומתי — אין אחריותיות (accountability).
הסבות ידניות תהליך ההסבה לנהג מנוהל ידנית במייל/נייר, ללא מעקב סטטוס מסודר.

3 · מצב חדש — המערכת שנבנתה

FineTracker — מקור אמת יחיד

אפליקציית ווב פנים-ארגונית ב-.NET 8 שהחליפה את ה-Excel — מעקב, תיעוד ובקרה מלאים על כל דוח תנועה וההסבה שלו.

מקור אמת יחיד כל הדוחות במערכת אחת — סטטוס עדכני ונראות מלאה בכל רגע.
Workflow מובנה תהליך בן 8 שלבים מבוסס-תפקידים — מקבלה ועד סגירה, כולל הסבות.
RBAC 3 תפקידים: ConversionOfficer · SystemAdmin · ManagerViewer.
Audit Trail מלא תיעוד כל שינוי סטטוס — מי, מה ומתי (accountability מלא).
Soft Delete דוחות לא נמחקים פיזית — שחזור ל-SystemAdmin בלבד.
ליבה יציבה .NET 8 · EF Core 8 · SQL Server 2022 · Docker.

5 · סביבה פנימית

טכנולוגיות ואתגרים בהעלאה לשרתי IAI

טכנולוגיות נדרשות
  • Docker / Kubernetes פנימי (AKS Edge)
  • SQL Server 2022 · אחסון Object פנימי (S3-תואם / MinIO)
  • Artifactory פנימי (NuGet feed) · Self-hosted fonts
  • Entra ID דרך App Proxy / Private Link
אתגרים ורגולציה
  • רגולציה פנימית — אישור סיווג מידע להעלאה לשרתים
  • סביבה Air-Gapped — אין הורדות, בנייה offline בלבד
  • סקירת אבטחה + הקשחה (non-root, HEALTHCHECK)
  • אחסון תמונות/מסמכים — נפח, גיבוי, הרשאות גישה

ההעלאה לשרתי IAI אינה רק טכנית — היא כפופה לתהליך אישור ורגולציה פנימית לפי סיווג המידע של דוחות ומסמכי תאונה.

6 · סביבה חיצונית

האם Azure של תע"ש יכול להיות הפתרון?

אם ל-IAI כבר יש סביבת Azure חיצונית מנוהלת — היא יכולה לארח את FineTracker כדייר, כולל אחסון הקבצים.

כן — כדייר בסביבה קיימת אם קיים Azure חיצוני של תע"ש, אפשר לפרוס לשם את המערכת בלי תשתית חדשה.
אחסון קבצים = Blob Storage מסמכי תאונה ותמונות ב-Azure Blob — עם גישה מאובטחת (SAS / Private Endpoint).
תנאי סף מיקום נתונים מאושר · הזדהות Entra ID · SLA · אישור סיווג מידע.
המסקנה: כן — בכפוף לסיווג המידע. סביבת Azure חיצונית של תע"ש היא המסלול המהיר ביותר לאירוח + אחסון קבצים, כל עוד דוחות ומסמכי תאונה מסווגים כמידע מנהלי (לא ביטחוני).
